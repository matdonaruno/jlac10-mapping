# エラー経験ナレッジベース設計

## 概要

過去のSNエラー対応実績を蓄積し、新しいエラー発生時に
類似事例をヒントとして提示する。

## データ構造

```json
{
  "knowledge": [
    {
      "id": "K001",
      "error_type": "JLAC10未設定",
      "item_name": "総蛋白",
      "item_code": "1001",
      "hospital": "H100",
      "vendor": "NEC",
      "symptom": "OBX-3が空でDB取り込みエラー",
      "cause": "電子カルテの検査マスターにJLAC10が未登録",
      "resolution": "3A010000002327101 を設定",
      "jlac10": "3A010000002327101",
      "notes": "時間外検査は別コードだが同じJLAC10で対応可能",
      "tags": ["生化学", "NEC", "JLAC10未設定"],
      "resolved_at": "2026-04-18",
      "issue_number": 4123
    }
  ]
}
```

## 類似エラー検索ロジック

新しいエラー項目に対して過去のナレッジを検索:

1. **項目名あいまいマッチ** — 同じ検査項目の過去事例
2. **エラー種別マッチ** — 同じ種類のエラーの対応パターン
3. **ベンダーマッチ** — 同じ電子カルテベンダーの事例
4. **タグマッチ** — 関連タグでの横断検索

## 実装タイミング

SNエラーダッシュボードの実装が安定してから着手。
LocalStorage に knowledge キーで保存。
案件クローズ時にナレッジ登録を促す。

## 蓄積フロー

```
案件クローズ時:
  → 「ナレッジに登録しますか？」プロンプト
  → エラー種別、原因、解決方法を入力
  → タグ自動推定（検査区分、ベンダー、エラー種別）
  → LocalStorage + JSON エクスポートで共有
```
