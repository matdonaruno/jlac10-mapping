# JLAC10 検査コード符番支援システム

## 目的

病院の検査マスター符番業務を支援するシステム。

- 外注先3社（SRL・BML・LSI メディエンス）の検査情報を自動収集
- JSLM JLAC10 コード表を月次取得
- 試薬添付文書から測定法を抽出
- 病院 SOP から測定法・試薬情報を抽出
- 上記を統合した JLAC10 データベースを構築
- 病院マスター（数千項目）に対して JLAC10 コードを自動マッピング

## JLAC10 コード構造

```
外注(15桁):  3A010 0000 023 271
             ├───┘ ├──┘ ├─┘ ├─┘
             │     │    │   └── 測定法コード（3桁）
             │     │    └────── 材料コード（3桁）
             │     └─────────── 識別コード（4桁）
             └───────────────── 分析物コード（5桁）★検索キー

院内NCDA(17桁): 上記15桁 + 結果識別コード（2桁）
```

- 外注（SRL/BML/LSI）は **15桁**（結果識別なし）
- 院内（NCDA）は **17桁**（結果識別2桁を付加）
- 結果識別は「共通コード」と「固有コード」の2種類

## CLI コマンド一覧

| コマンド | 説明 |
|---------|------|
| `srl-scraper srl` | SRL 全カテゴリ取得 |
| `srl-scraper bml` | BML 全検査項目取得 |
| `srl-scraper lsi` | LSI 全検査項目取得 |
| `srl-scraper jslm` | JSLM JLAC10 コード表取得 |
| `srl-scraper merge` | 3社+JLAC10マスターを統合 |
| `srl-scraper search [QUERY]` | あいまい検索（省略で対話モード） |
| `srl-scraper reagent` | 試薬DB構築（メーカーサイト取得） |
| `srl-scraper reagent --pmda URL` | PMDA添付文書を手動追加 |
| `srl-scraper sop FILE_OR_DIR` | SOP パース（Word/PDF） |
| `srl-scraper check` | 全ソースの更新状況確認 |
| `srl-scraper diff OLD NEW` | 2つの JSON の差分比較 |
| `srl-scraper list` | SRL カテゴリ一覧 |

**共通オプション:**

| オプション | 説明 |
|-----------|------|
| `--check-update` | 更新があるときだけ取得 |
| `--no-cache` | キャッシュを無視して再取得 |
| `--cache-age N` | キャッシュ有効期限を N 時間に変更 |
| `-o DIR` | 出力ディレクトリ指定 |
| `-n N` | search の最大結果数 |
| `-v` | 詳細ログ出力 |

## サーバー負荷対策

| 対策 | 詳細 |
|-----|------|
| リクエスト間隔 | 全サイト共通 3秒 |
| HTML キャッシュ | 24時間有効、2回目以降はサーバーアクセスなし |
| 更新検知 | トップページ1回で更新有無判定、変更なしなら即終了 |
| リトライ | 5秒→10秒の指数バックオフ、最大3回 |
| PMDA | 自動巡回禁止（利用規約）、手動URL指定のみ |


### 一括マスターマッピング（TODO）

```
病院マスター(Excel/CSV)
  ↓ JSON変換
  ↓ 各行に対してあいまい検索
  ↓ SOP/試薬情報で測定法を絞り込み
  ↓
┌─────────────────────────────────────┐
│ 自動確定 (スコア≥90, 候補1件)        │ → そのまま採用
│ 候補提示 (スコア≥50, 候補複数)       │ → 手動で選択
│ 手動対応 (スコア<50 or 候補0件)      │ → 手動で調査・入力
└─────────────────────────────────────┘
  ↓
マッピング結果JSON
  ↓
確定分を共有DBに差分還元
（新たな「院内名称→JLAC10」対応として蓄積）
```

### DB還元の仕組み

病院Aで「TP」→ `3A0100000023271` と確定
→ merged DB に「院内名称: TP」が追加される
→ 次の病院Bで「TP」を検索したとき、この実績がスコアに反映
