## 本書の構成（原則: 付番メソッドと出力器を分離）

- **Part A: 付番メソッド** — 病院マスタから JLAC10 / JANIS を導出するロジック（SOP §2.3 + 原本コード表から派生）
- **Part B: 出力器** — マッピング結果を設定依頼書式に落とし込む器（18列NCDA標準、SSI Lookup、富士通JJ、NEC指定 等）
- **Part C: 実データ構造** — SSMIX2 OBR/OBX における細菌検査の実表現
- **Part D: データ資産の JSON 管理**
- **Part E: Pages UI の検体/細菌 分離方針**

> ⚠️ 18列フォーマットは **マッピング結果を入れる器**。ここから付番メソッドは読み取れない。付番は SOP と原本コード表から導出する。

### A.6 結果識別コード（2桁）— 2レイヤ構造

原本 `data/136jlac10_1.xlsx` は「共通コード」と「分析物固有コード」の2シートを持つ。

#### A.6.1 共通コード（全分析物共通、01-49）

マスタ登録で使う主要コード:

| コード | 名称 | 用途（細菌検査） |
|---|---|---|
| `01` | 定量値 | 6B010 / 6B105 かつ 材料 = `001`(尿含む) の菌量 |
| `05` | 希釈倍率 | 薬剤感受性の MIC値（結果2） |
| `11` | 判定 | 6B305 液体培地法(`747`) の結果2 |
| `12` | スコア | 菌量（+/++/+++、Gaffky号数）（結果2） |
| `14` | 型 | 菌名 / 薬剤名 / 型・パターン判定（結果1） |
| `49` | 別紙報告 | 依頼と結果が1:1で紐付かない場合の**依頼側** |

ペアの原則（SOP 段落374）:
- 依頼と結果が 1:1 で紐付く場合、依頼側と結果側で結果識別コードを統一
- 1:1 で紐付かない場合、依頼側に `49`（別紙報告）

#### A.6.2 分析物固有コード（51-99、分析物ごとに意味が異なる）

**6C010 薬剤感受性検査(一般細菌)** の例（薬剤×MIC/判定ペア）:
- 51 アミカシン MIC値 / 52 アミカシン 判定
- 53 アモキシシリン MIC値 / 54 アモキシシリン 判定
- ...
- 奇数 = MIC値、偶数 = 判定 のペアで薬剤ごとに割当

**6B652 FilmArray血液培養パネル**（遺伝子検査）:
- 00 核酸同定パネル総合
- 51 Enterococcus属 / 52 Listeria monocytogenes / 53 Staphylococcus属 ...
- 各菌種/耐性遺伝子に連番で割り当て

**5E110 クロストリジオイデス・ディフィシル抗原**（細菌検査の免疫学系統）:
- 51 抗原(GDH) / 52 毒素

#### A.6.3 マスタ登録時の選択指針

- 塗抹培養・薬剤感受性の **汎用マスタ**（総称項目）: 共通コード `14` / `12` / `05` / `01` / `11` を使う
- 遺伝子検査パネル / 個別薬剤ごとのマスタ: 固有コード `51-99` を使う（Multi-parameter gene-related testing.xlsx 参照）

#### A.7.1 病院マスタ表記ゆれへの対応（運用上の実態）

病院マスタは人手入力のため **タイポ・全角スペース・半角カナ・濃度付記** 等で揺れる。**病院の責任範囲であり修正しない**。代わりに:

- `NCDA/JANIS菌名縦積み.xlsx` に過去の表記ゆれを積み上げ続ける（14,809行 / 554ユニークコード、平均27通り/コード）
- 最頻出は `9999` に 2,637通り、`9998` に 1,085通り（分類不能例の履歴）
- 新規表記が解決したら縦積みに追記して履歴資産化

**正規化マッチャ（パイプライン）:**
1. 完全一致検索（縦積みJSON）
2. 正規化（全角→半角、半角カナ→全角、連続スペース圧縮、先頭マーカー `*`/`★` 除去、末尾ピリオド除去、濃度数値剥ぎ）→ 再検索
3. 属/種分解フォールバック（`Genus species` 部分一致）
4. SOP §2.3 ③-2 ルール適用（属名 / 9998 / 9999）
5. 人手確定した新規表記を縦積みに追記

## Part C. 実データ構造（SSMIX2 OBR/OBX）

### C.1 セグメント概観

```
#SSMIX,2.00,0,0,...,OML-11,...
SPM|0001|||043^腹水^JC10^050544^腹水^99Z13|        ← 検体(材料)
OBR|0001|||6A0100000043704^塗抹検査^JC10           ← 検査依頼
OBX|0001|CWE|6A010000004370414^塗抹菌名^JC10^302003^塗抹菌名^99ZS6|1|1014^グラム陰性桿菌^JANIS^302003^...  ← 結果1(菌名+JANIS)
OBX|0002|ST |6A010000004370412^塗抹菌量^JC10^302003^塗抹菌量^99ZS6|1| 少数,貪食：疑い                     ← 結果2(菌量)
```

- **SPM**: 検体（材料）情報
- **OBR**: 検査依頼（JLAC10 = 分析物+識別+材料+測定法+`704` 塗抹 or `741` 培養 等）
- **OBX**: 結果の繰り返し。第3フィールドに JLAC10、値フィールドに結果値（菌名文字列 / 菌量値 / JANISコード）

### C.2 固定 / 変動

| 要素 | 扱い |
|---|---|
| JLAC10 17桁（分析物/識別/材料/測定法/結果識別） | **固定**（マスタに定義） |
| JANIS コード（菌固定マスタの場合） | **固定** |
| 菌名文字列 | **変動**（結果値） |
| 菌量値（+, ++, 少量, 10^7 cfu/mL 等） | **変動** |
| 薬剤名・薬剤数（検出菌に応じて） | **変動** |
| MIC値 | **変動** |

### C.3 1オーダー = 複数 OBX

1検査オーダー（OBR）に対して、**検出菌 × 検査項目** の数だけ OBX が並ぶ。例:

- 塗抹: 菌名(14) + 菌量(12) の2 OBX（菌1種につき）
- 培養: 菌名(14) + 菌量(12) の2 OBX（検出菌ごと、3菌なら6 OBX）
- 薬剤感受性: 薬剤名(14) + MIC(05) の2 OBX（薬剤ごと、検出菌×薬剤セット分）

菌ごとの薬剤セットは異なる（菌Aで検査する薬剤と菌Bで検査する薬剤は別）。

### C.4 マスタ運用の併存パターン

| タイプ | 院内コード例 | マスタのJANIS欄 | 結果値での菌名 |
|---|---|---|---|
| 菌固定マスタ | `304088` Campylobacter ureolyticus | 固定値（4桁）あり | 省略可（マスタ参照） |
| 総称マスタ | `305055` 培養同定菌名 | 空 | 結果値で都度報告 |

両者は病院ごとに併存する運用差であり、設計原則の分岐ではない。18列フォーマットの列8(JANIS)は前者で埋まり、後者で空になる。

## Part E. Pages UI の 検体/細菌 分離方針

### E.1 基本方針

検体検査と細菌検査で **明確に処理パイプを分離**する（Import / Bulk Map / SN Error の各タブで「検体モード」「細菌モード」を持つ）。Bulk も別。

### E.2 リニア遷移

両方処理したい場合のため、**片方完了 → もう片方へ進む導線**を用意。運用ケース:
- 病院マスタ一括投入時、検体検査と細菌検査が混在する → 分類後に検体→細菌の順でリニア処理
- SN Errorでは、検体エラーと細菌エラーを分類表示してそれぞれのフローに流す

### E.3 UI要素

- 検体モード: 既存の Import/Bulk/SN Error UI 流用
- 細菌モード: 上記パイプ（Part A）を UI 化。検査種別ラジオ（迅速/塗抹培養/薬剤感受性/遺伝子）、JANIS マッチ結果表示、縦積み追記ボタン
- リニア遷移バー: 「検体 3件完了 → 細菌 5件処理中 → 細菌結果をレビュー」
