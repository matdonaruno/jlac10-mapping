# SSMIX2 エラー対応ワークフロー

## 概要

病院の電子カルテから SSMIX2 標準化ストレージにデータ出力する際、
DB 取り込み前チェックでエラーが発生する。
エラーの多くは JLAC10 コード未設定・不正によるもの。
マッピング後は病院への設定依頼、設定確認、出力検証まで行う。

---

## SSMIX2 データ構造

```
SSMIX2 標準化ストレージ:
  HL7v2.5 メッセージをファイルとして保存

  ファイル名パターン:
    {患者ID}_{日時}_{メッセージ種別}_{イベント種別}_{オーダーID}

  検査関連メッセージ:
    OML^O33  検体検査オーダー（依頼）
    OUL^R22  検体検査結果

  主要セグメント:
    MSH  メッセージヘッダー
    PID  患者情報
    OBR  検査オーダー情報
    OBX  検査結果（1項目1セグメント）
      OBX-2: データ型（NM=数値, ST=文字列, CE=コード等）
      OBX-3: 検査項目コード（JLAC10）
      OBX-5: 結果値
      OBX-6: 単位
```

## エラーの主な原因

| エラー種別 | 原因 | 対応 |
|-----------|------|------|
| JLAC10 未設定 | OBX-3 が空 or 無効なコード | マッピングツールで符番 |
| JLAC10 形式不正 | 桁数不足、不正文字 | コード修正 |
| 材料コード不正 | JLAC10 の材料部分(10-12桁)が無効 | コード確認・修正 |
| 結果値形式不正 | OBX-2(データ型)と OBX-5(値)の不整合 | データ型を確認 |
| 必須セグメント欠落 | OBR/OBX が不足 | 電子カルテ側設定確認 |
| 文字コード問題 | エンコーディング不正 | 出力設定確認 |

## 各フェーズの詳細

### PHASE 1: エラー検出・起票

**実行者:** エラー対応担当者
**ツール:** GHE Issues + Rocket.Chat
**手作業:** イシュー作成、チャット連絡

### PHASE 2: マッピング

**実行者:** JLAC10 マッピング担当
**ツール:** JLAC10 MAPPER (Pages/CLI)

```powershell
# エラー項目を検索
py -m uv run srl-scraper search "エラー項目名"

# 一括で分析（エラーリストがある場合）
py -m uv run srl-scraper -v map error_items.xlsx --col-name B

# 符番結果を DB に蓄積
py -m uv run srl-scraper apply-mapping mapping_result.json --hospital H100
```

Pages SN Error タブでも対応可能:
- Excel ファイル or テキスト貼り付け（HL7v2 OBX 自動検知）
- 付番可能/要問い合わせに自動分類

### PHASE 3: 病院への設定依頼

**実行者:** JLAC10 マッピング担当
**ツール:** Outlook（メール送信）、GHE Issues、Rocket.Chat

**メール構成:**

| 要素 | 内容 |
|------|------|
| 宛先 | Outlook 連絡先の病院担当者 |
| CC | 関係者 |
| タイトル | `{病院名}_{発生日時}_#{イシュー番号}` |
| 本文 | 挨拶 → エラー発生通知 → マッピング結果表 → 設定依頼 → 締め |

**メール送信後のアクション:**
1. GHE イシューにコメント: 「マッピング完了、病院へ設定依頼済み」
2. Rocket.Chat に記述:
   - 同じ内容
   - GHE イシューコメントのリンク
   - 関係者へのメンション

### PHASE 4: 病院からの設定完了連絡

**トリガー:** 病院担当者からのメール（設定完了報告）
**アクション:**
1. GHE イシューにコメント: 「病院側設定完了」
2. Rocket.Chat に同様のコメント

### PHASE 5: 修正確認（出力検証）

**トリガー:** 病院からの SSMIX2 再出力
**アクション:**
1. 再出力データのエラーチェック
2. OK → イシュークローズ + 完了報告
3. NG → PHASE 2 に戻る

## イシューのライフサイクル

```
Open（起票）
  │
  ▼
In Progress（マッピング中）
  │
  ▼
Waiting（病院設定待ち）
  │
  ▼
Verifying（出力確認中）
  │
  ├── OK → Closed（解決）
  └── NG → In Progress に戻る
```

## 自動化計画

### 現在手作業のもの → 自動化可能性

| 手作業 | 自動化方法 | 必要な確認 |
|--------|-----------|-----------|
| イシュー作成 | Pages → GHE API | GHE API 利用可否 |
| 設定依頼メール作成 | Pages でメール本文生成 → コピー | テンプレート確定 |
| GHE コメント追加 | GHE API | GHE API 利用可否 |
| Rocket.Chat 投稿 | Rocket.Chat API/Webhook | API 利用可否 |
| 出力検証 | SSMIX2 データ自動チェック | エラーログ形式確認 |

### Phase 1: メール・コメント生成の半自動化（API不要）

```
Pages SN Error タブで:
  1. エラー分析完了
  2. 「Generate Email」ボタン → メール本文マークダウン生成 → コピー
  3. 「Generate Issue Comment」ボタン → GHE コメント用テキスト → コピー
  4. 「Generate Chat Message」ボタン → Rocket.Chat 用テキスト → コピー
```

### Phase 2: GHE API 連携（API確認後）

```
Pages から直接:
  - イシュー作成
  - コメント追加
  - ステータス変更
```

### Phase 3: Rocket.Chat 連携（API確認後）

```
Pages から直接:
  - チャンネルにメッセージ投稿
  - メンション付き
  - イシューリンク付き
```
