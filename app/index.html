User Guide

SEARCH

検査項目名、略称、英名であいまい検索します。
例: TP, ALB, CRP, HbA1c, アルブミン, albumin
結果は SRL/BML/LSI 3社の情報付きで表示。
スコア: 100=完全一致, 95=略称, 80+=先頭一致, 60+=部分一致

IMPORT

過去マッピング結果の Excel を JSON に変換し DB に統合。
1. 病院名を入力
2. Excel ファイルを選択
3. 列番号を設定（A=項目名, B=略称, C=JLAC10, D=正式名称）
4. Convert → Merge to DB → Export Merged DB

BULK MAP

病院マスター Excel を一括でJLAC10 にマッピング。
1. Excel ファイルと項目名列を指定
2. Map All で全件自動検索
3. OK=自動確定(90+), ??=候補あり(50+), --=手動
4. ??の行はプルダウンで候補を選択
5. Confirm → Merge to DB で蓄積
6. Download Result Excel で結果保存

SN ERROR

SSMIX2 の SN エラー項目を分析。
Excel ファイルまたはテキスト貼り付けで入力。
付番可能な項目と要問い合わせ項目を自動分類。
問い合わせメールを自動生成 → コピーして送信。

DATABASE

DB はページ読み込み時にリポジトリから自動ロード。
手動で別の JSON をロードすることも可能。
病院 JSON を追加で読み込んでマージ可能。
Export Merged DB でダウンロード → git push で共有。

JLAC10 CODE STRUCTURE

3A010-0000-023-271-**
5桁:分析物 | 4桁:識別 | 3桁:材料 | 3桁:測定法 | 2桁:結果識別
外注(SRL/BML/LSI) = 15桁（結果識別なし → **）
院内(NCDA) = 17桁（結果識別あり）

WORKFLOW

1. DB は定期更新済み（SRL/BML/LSI/JSLM）
2. Search で単発検索 or Bulk Map で一括処理
3. 確定したマッピングを DB に蓄積
4. 蓄積するほど次回の精度が上がる
5. Export → git push で全員に共有
